# eDSL + コード生成によるモデル定義と自動ヤコビアン — 設計

## 動機

現在のモデル定義方法（手動でPropagationModel trait実装）は型安全だが、
EKF用のヤコビアンを得るには有限差分（遅い）か手動実装（面倒）が必要。

目標: **構造体ベースの型安全性を維持しつつ、ビルド時にヤコビアンを自動生成する**。

## アーキテクチャ

```
ユーザーが build.rs で記述:
  ModelBuilder::new("OrbitalPropagation")
    .state_field("position", 3)   // FrameVec<Eci, Length, Kilo>
    .state_field("velocity", 3)   // FrameVec<Eci, Velocity, Kilo>
    .propagation(|s, dt| vec![
        s[0] + s[3] * dt,  // x' = x + vx * dt
        s[1] + s[4] * dt,
        ...
    ])
    .generate("src/generated/orbital.rs");
          ↓ ビルド時に実行
  src/generated/orbital.rs:
    - PropagationModel impl（forward関数）
    - orbital_propagation_jacobian()（解析的ヤコビアン）
    - 必要な型のuse文
```

## コンポーネント

### 1. 記号式 AST (`Expr`)

```rust
enum Expr {
    Var(usize),         // 状態変数 x[i]
    Param(String),      // パラメータ (dt, mu, etc.)
    Const(f64),         // 定数
    Add(Box<Expr>, Box<Expr>),
    Mul(Box<Expr>, Box<Expr>),
    Div(Box<Expr>, Box<Expr>),
    Neg(Box<Expr>),
    Sin(Box<Expr>),
    Cos(Box<Expr>),
    Sqrt(Box<Expr>),
    Pow(Box<Expr>, i32),
}
```

### 2. 記号微分 (`diff`)

```rust
fn diff(expr: &Expr, var: usize) -> Expr {
    // 微分規則の適用 (連鎖律、積の微分、etc.)
}
```

### 3. 簡約化 (`simplify`)

```rust
fn simplify(expr: &Expr) -> Expr {
    // 0 * x → 0, 1 * x → x, x + 0 → x, etc.
}
```

### 4. コード生成 (`to_rust`)

```rust
fn to_rust(expr: &Expr, state_var: &str, params: &[&str]) -> String {
    // Expr → "state[0] + state[3] * dt" のようなRustコード文字列
}
```

### 5. ModelBuilder (ユーザーAPI)

build.rs で使用。記号式で伝搬・観測モデルを記述し、
`.generate("path/to/output.rs")` で .rs ファイルを出力。

## 生成されるコードの例

入力:
```rust
// build.rs
let mut m = ModelBuilder::new("Linear");
m.state_field("position", 3);
m.state_field("velocity", 3);
let s = m.state_vars();
let dt = m.param("dt");
m.propagation(vec![
    s[0] + s[3] * dt, s[1] + s[4] * dt, s[2] + s[5] * dt,
    s[3], s[4], s[5],
]);
m.generate("src/generated/linear.rs");
```

出力 (src/generated/linear.rs):

## 選択肢

### A. build.rs + codegen ライブラリ (採用)

ユーザーの build.rs から呼び出す。出力は .rs ファイル。

- **Pros**: 生成コードが可読。ビルド時に生成。型安全。
- **Cons**: build.rs の依存が増える。

### B. proc macro eDSL

`model! { ... }` マクロ内に記述。

- **Cons**: 生成コードが不可視（cargo expand でのみ確認可能）。
  ユーザーの明示的な要望に反する。

### C. 外部ツール (Python + sympy 等)

- **Cons**: Rust エコシステム外の依存。ポータビリティ低下。

## 制限事項（初期版）

- ユークリッド状態のみ（Vector3, f64）。多様体（quaternion, direction）の
  接空間微分は手動 Jacobian と組み合わせ可能。
- 三角関数、平方根、べき乗は対応。
- 行列演算はスカラー展開で表現。
